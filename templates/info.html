SIRCODI

Sistema de Registro y Control para Personas con Diabetes

INFORMACIÓN:

Hola, bienvenid@!... SICORDI es una Aplicación Web o lo que hoy en día se conoce como "WebApp".

Como bien lo aclara el título, es un sistema de control y registro para aquellas personas con Diabetes. El Registro se realiza a través de un formulario donde se le pedirá al usuario o paciente ingresar los siguientes datos: Nombre Completo, Edad, DNI, Sexo Biológico y finalmente el Nivel de Azúcar en Sangre. Cabe aclarar que la medición del Nivel de Azúcar se debe realizar en Ayunas mediante el "Glucómetro" o simplemente, Medidor de Glucosa.

Al ser una WebApp, como tal, brinda información para desarrolladores, convirtiendola, además, en una API a la cual se le puede consumir y/o extraer información útil para quien la necesite.

Para re-generar la Base de Datos deberá el cliente estar logueado como administrador, que deberá ingresar el nombre de usuario y su clave numérica. En este caso, y, por el momento sólo el administrador del servidor tiene acceso a dichos datos.

También permite mostrar gráficos y tablas de la información y registro que va realizando cada paciente, permitiendo así un mayor control y visualización tanto para el mismo como para el médico y/o diabetólogo a cargo. Más abajo hay algunas secciones que permiten un mejor descripción de cada parte del programa.

NOTA:

Quiero aclarar en 1er lugar que dicha WebApp, en estos momentos, está diseñada con fines meramente académicos, lo que no quiere decir, que en un futuro y luego de seguir siendo desarrollada, pueda ser adaptada con otros fines. Debido a lo mencionado, y, por cuestiones de seguridad y para mantener a la persona en forma anónima el DNI no será mostrado ni en la API, ni en tablas y/o gráficos.

Otro de los objetivos principales, y a futuro, es que pueda ser útil para llevar una mayor fluidez para los controles y comunicación entre pacientes y médicos sin siquiera tener un contacto físico.

Por último aclarar que la página aún se encuentra en desarrollo. Ante cualquier consulta en el pie de la página se encuentra la información para contactarme.

API PARA DESARROLLADORES:

Esta sección es para aquellos desarrolladores/programadores que quieran realizar prácticas consumiendo la información que brinda esta WebApp para fines académicos.

La información brindada está en formato JSON y cuenta con todos los registros realizados por cada paciente y/o cliente. Los campos que posee esta API son: {id, age, datetime, value} siendo: el id: id del paciente, age: la edad, value: el valor de azúcar en sangre y, datetime: la fecha y hora que registró dicho valor.

Para acceder al dataset puede ingresar en API sino, también copiando el siguiente endpoint: "/niveles/api"" y pegandolo a continuación de la página. Ej: http//.../niveles/api

Dicho endpoint permite el ingreso opcional de una query string: '?key=value' que se coloca seguido del endpoint: http//.../niveles/api?key1=value1&key2=value2&key3=value3. Ejemplo:

id: http//.../niveles/api?id=1 muestra los registros del paciente cuyo id corresponda.

limit: http//.../niveles/api?id=1&limit=1 o http//.../niveles/api?limit=1 muestra la cantidad máxima de registros a mostrar.

offset: http//.../niveles/api?id=1&limit=1&offset=1 o http//.../niveles/api?limit=1&offset=1 o http//.../niveles/api?offset=1 muestra los registros pero ahora a partir del valor que tome el offset.

GRÁFICOS:

La persona cuyo formulario haya completado y enviado a la Base de Datos tendrá la opción de ver un gráfico de línea de como estuvo variando su nivel de azúcar. Esto también es aplicable para el médico en cuestión ya que tiene información gráfica del registro de sus pacientes.

Para acceder al gráfico de los registros personales debe colocar el endpoint '/niveles/ingresar_dni_persona/grafico' seguido del link de la página. Ejemplo: https://.../niveles/33456789/grafico

Otro tipo de gráfico que presenta esta WebApp es el de barras, que realiza una comparativa de la cantidad de personas que han hecho un registro según grupo etario, y, según el sexo biológico de cada uno. Para acceder al gráfico comparativo usar el siguiente endpoint: '/comparativa'. Ejemplo: https://.../comparativa

TABLAS:

La persona cuyo formulario haya completado y enviado a la Base de Datos tendrá la opción de ver una tabla de como estuvo variando su nivel de azúcar. Esto también es aplicable para el médico en cuestión ya que tiene información en formato de tabla del registro de sus pacientes como así también el promedio y la cantidad de registros de mediciones realizadas.

Para acceder a la tabla de los registros personales debe colocar el endpoint '/niveles/ingresar_dni_persona/tabla' seguido del link de la página. Ejemplo: https://.../niveles/33456789/tabla

Esta WebApp también posee una tabla con todos los registros realizados de todas las personas que hayan completado el formulario. Útil para tener otra forma de visualización de los mismos. Para acceder a la tabla usar el siguiente endpoint: '/niveles/tabla'. Ejemplo: https://.../niveles/tabla

Dicho endpoint permite el ingreso opcional de una query string: '?key=value' que se coloca seguido del endpoint: http//.../niveles/tabla?key1=value1&key2=value2&key3=value3. Ejemplo:

id: http//.../niveles/tabla?id=1 muestra los registros del paciente cuyo id corresponda.

limit: http//.../niveles/tabla?id=1&limit=1 o http//.../niveles/tabla?limit=1 muestra la cantidad máxima de registros a mostrar.

offset: http//.../niveles/tabla?id=1&limit=1&offset=1 o http//.../niveles/tabla?limit=1&offset=1 o http//.../niveles/tabla?offset=1 muestra los registros pero ahora a partir del valor que tome el offset.
